Cellar Website (Customer Ordering and Employee Management)

I. Introduction

Our Cellar website has two parts to it. We want to create an online ordering system for students who do not want to call in. Additionally, we want a way for employees to view what orders are in the queue as well as ingredient stock and other store management information. We want a way to store every menu item, the ingredients that they require, and the time that it takes to make each item. We want a way to display what time orders are sent in and an approximation of when they will be done. Other functionality that we want to include is employee efficiency ratings and what ingredients are still in stock. All of these cases would require a database.

II. Use Cases

  • Use Case 1: (Evan Lebakken)
    Cellar customers can place an order for any food item on the menu that is in stock, and that order gets associated with the student's account as is put on the order queue.

  • Use Case 2: (Bradley Olson)
    Cellar employees can see the entire order queue, ordered chronologically.

  • Use Case 3: (Nalin Richardson)
    Students can create an account, entering a username and password as well as basic information such as student ID, first name, and last name.

  • Use Case 4: (Nalin Richardson)
    Students and employees can log in using a username and password, which are encrypted in the database.

  • Use Case 5: (Evan Lebakken)
    Other customers can place orders via a guest form that takes in all required fields as well as first and last name to construct an order.

  • Use Case 6: (Patrick Wigger)
    On the employee facing page, employees can report low levels of ingredients. This will affect what customers are able to order.

  • Use Case 7: (Bradley Olson)
    Employees can associate their employee ID with an order, and mark the order as completed when done.

  • Use Case 8: (Bradley Olson)
    An ingredients stock page shows the ingredients which are low in stock, allowing for easier re-stocking.

  • Use Case 9: (Nalin Richardson)
    Students with accounts can see all past orders, re-order past orders, and give feedback on past orders.

  • Use Case 10: (Patrick Wigger)
    Feedback will be measured in stars and will include a review section for text, and each feedback will be associated with the employee. Employees will have a calculated star value viewable by them and their managers.

  • Use Case 11: (Patrick Wigger)
    Managers have a hub page showing all working employees, the order queue, and the status of the ingredient stock.

  • Use Case 12: (Evan Lebakken)
    The student order page contains an "I'm feeling lucky" button that orders a random pizza. The daily, weekly, and monthly most popular orders are also shown as available orders.

III. Project Management Plan

Shared code will be managed my linking the home directory of our project on our remote server to Github. Each team member can then clone the project to their own machine and push updated to Github as well as pull and test code on the remote server.

Communication will be managed through a combination of Slack messages, text messages, and in-person meetings. The in-person meetings will take place at least on Fridays of each week and should be where the majority of project work occurs.
