1. The command you type into console to check if apachectl is running is 'sudo apachectl status or fullstatus'.

2. GET is a simpler and faster version of POST. POST should be used when the server or database needs to be updated, if you have a large amount of data (no size limitation) and for sending user input. HEAD just returns the header of a resource rather than the entire resource.

3. A ServerRoot is the top of the directory tree that contains server's log, config, and error files. A DocumentRoot is the directory documents are served out of, and by default a DocumentRoot takes all requuests

4. The web browser listens on port 80 and port 443

5. The directory apache serves documents from is '../var/www/html'.

6. '/var/log/apache2/access.log/ contains all of Apache's traffic logs. '/var/log/apache2/error.log' contains Apache's error logs. To see the most recent events at the end of the file, use 'tail' command before the directory

7. A directory index file is what is served if a directory is requested. It would be nice to have one in each directory because it means that all the information is not all stored in one file.

8. We acheived this through a series of commands within the command line.

> sudo a2enmod userdir

> sudo nano /etc/apache2/mods-enabled/userdir.conf

Add this to the file:

"IfModule mod_userdir.c>

UserDir public_html

UserDir disabled root

Directory /home/*/public_html>

AllowOverride All

Options MultiViews Indexes SymLinksIfOwnerMatch

Limit GET POST OPTIONS>

# Apache <= 2.2:

# Order allow,deny

# Allow from all

# Apache >= 2.4:

Require all granted

/Limit>

LimitExcept GET POST OPTIONS>

# Apache <= 2.2:

# Order deny,allow

# Deny from all

# Apache >= 2.4:

Require all denied

/LimitExcept>

/Directory>

/IfModule>"

> sudo service apache2 restart

> mkdir /home/$USER/public_html



9. You create a password protected directory by taking three steps. First you change the file apache2.conf so that the line reading 'AllowOverride none' instead reads 'AllowOverride all'. Then you create the file '/etc/apache2/.htpasswd', and fill it with username:password pairs using the command 'sudo htpasswd -c /etc/apache2/.htpasswd username' multiple times, but without the -c after the first time. Thirdly you create a .htaccess file in the directory you want to protect and add these lines to the file:
AuthUserFile /etc/apache2/.htpasswd
AuthType Basic
AuthName 'Restricted Content'
Require valid-user
